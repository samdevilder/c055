The key multilingual issues that the W3C address within the internationalization activity can be summarized as any barriers of deployment and use of the Web and its technologies due to either language, script, region or culture. Within this Internationalization activity there are three major groups known as the International Working Group (IWG), the International Interest Group, and the Internationalization Tag Set (ITS) Interest Group, each group serving a different role within the activity.

    It is the IWG that produces content and documents. These include requirements, guidelines and proposals for specification writers, as well as educational resources, anything to help developers and content creators to better understand and more widely implement the Internationalization aspects of W3C technology. Under their Activity Project RADAR page (http://w3c.github.io/i18n-activity/projects/) we will find that the projects fall under categories Requirements development, Developer guidelines, Reviews, discussion, advice, Implementation support, Education and outreach. All the projects run by the Internationalization activity aim to make the Web more universal. To break down all language, cultural and region barriers on the World Wide Web.

  6. http://www.few.vu.nl and http://cs.vu.nl both share the ip adress 130.37.164.154. We found this by using the following website: www.site24x7.com. Sharing the same IP address can reduce costs for most businesses and institutions with exceptions for corporate giants like Google (a company with multiple data centers). For most companies and educational institutes, to have multiple dedicated servers for each branch or course that they offer would be impractical and cost inefficient since the costs would outweigh the benefits they would gain from the multiple dedicated servers. Another issue regarding a server's functionality is security. Since both http://www.few.vu.nl and http://www.cs.vu.nl share an IP address, if there were to be a DDoS attack on either of these sites, all other sites sharing this IP address with them would also be out of service until the attack is resolved.

  7. Internet Protocol version 6 (IPv6) is a much more advanced protocol and has better features than IPv4. Designed to be an evolutionary upgrade that will allow steady growth over time both in terms of Internet traffic and number of hosts connected. It has an infinite number of addresses. It will replace IPv4 world to solve the growing number of networks and the shortage of IP addresses. The number of bits in an IPv4-address are 32 in format decimal equating to a possible 4.3 billion number of addresses. An IPv6-address contains 128 bits in a hexadecimal format which equates to an infinite possible number of addresses. IPv6 will also make some important technological changes that were issues under IPv4. Changes such as auto-configuration, no more private address collisions, better multicast routing and more. http://www.webopedia.com/DidYouKnow/Internet/ipv6_ipv4_difference.html

  5. 'AJAX' - asynchronous JavaScript and XML. Asynchronous refers to a non-sequential I/O (input/output), a form of processing that allows other non-I/O-dependant processing to continue before transmission has finished. The simple approach to I/O would be synchronous I/O. While it is simple, it does leave a lot of system resources remaining idle while they wait for all processing to be completed. However in an asynchronous approach, any task that relies on the I/O being complete will still have to wait, but other processing that has no dependency on the I/O operation can continue.
